
## OSPF


OSPF(Open Shortest Path First)는 네트워크 장비들이 서로의 경로 정보를 주고받아<br/>
최적의 길을 자동으로 찾아내는 '동적 라우팅 프로토콜'입니다.<br/>

이전에 질문하신 RIP는 최대 15대까지만 연결할 수 있고 속도가 느려 대규모 망에서는 쓰지 않습니다.<br/> 
반면 OSPF는 규모가 큰 대기업이나 기관에서 가장 표준으로 많이 사용하는 프로토콜이기 때문에, <br/>
네트워크를 공부하신다면 반드시 익혀야 할 핵심 기술입니다.<br/>

R1(config)# router ospf 1<br/>
R1(config-router)# network 10.1.12.0 0.0.0.255 area 0<br/>
R1(config-router)# network 10.1.13.0 0.0.0.255 area 0<br/>
R1(config-router)# router-id 1.1.1.1<br/>
R1(config-router)#do clear ip ospf process <br/>
Reset ALL OSPF processes? [no]: y


R2~R4도 동일하게 해줍니다

R1(config-router)#passive-interface default<br/>
R1(config-router)#no passive-interface g0/0<br/>
R1(config-router)#no passive-interface g1/0<br/>

R1(config-router)#distance 85   ← eigrp보다 우선순위(선호도)가 앞서게 된다.<br/>
(eigrp 90)<br/>

ospf에 참여하는 모든 라우터의 네이버가 연결된 인테페이스에서 네트워크 타입을 변경한다.<br/>
R4(config)#int range f3/0 , g1/0<br/>
R4(config-if-range)#ip ospf network point-to-point <br/>

메트릭 계산<br/>

R3에서 아래와 같이 GIGA, FAST 인터페이스의 cost 가 동일하게 1로 잡혀있다. 즉 차이가 없다.<br/>
COST = Reference bandwidth / interface’s bandwidth<br/>
기본 Reference bandwidth: 100 mbps<br/>

router ospf 1<br/>
auto-cost reference-bandwidth 10000<br/>


COST = Reference bandwidth(100 -> 10000)  / interface’s bandwidth
