### cost를 수정하는 1번째 방법: ospf를 사용하는 인터페이스에 들어가서 cost를 직접 지정한다.

R1 Loopback 0

conf t<br/>
int l0<br/>
ip add 1.1.1.1 255.255.255.0<br/>
router ospf 1<br/>
net 1.1.1.1 0.0.0.255 area 0<br/>
exit<br/>

ip ospf cost 1000<br/>
end<br/>

sh ip int br(간단하게 cost 확인가능)<br/>
sh int g0/0(Full-duplex, 100MB/s, 1같이 수치 확인 가능)<br/>


### cost를 수정하는 2번째 방법: ospf를 사용하는 인터페이스에 들어가서 bandwidth를 수정한다.

int g0/0<br/>
bandwidth 1000000(1기가)<br/>
do sh int g0/0(Full-duplex, 1000MB/s, 1로 수치 바뀐거 확인 가능)<br/>


### DR/BDR 선발
Broadcast 네트워크 타입에서 DR/BDR을 선발한다. <br/>
Point-to-Point 에서는 선발하지 않는다.<br/>
DR/BDR을 선발하는 이유는 Broadcast 와 같은 다중접속 네트워크에서는 모든 라우터간에 LSA를 교환함으로써 네트워크가 복잡해지는데,
DR을 이용해서 LSA를 줄여준다.<br/>

#### R1 부터 R5까지 router-id 수정

conf t<br/>
router ospf 1<br/>
router-id 1.1.1.1<br/>
network 192.168.1.0 0.0.0.3 area 0<br/>

#### R5

conf t<br/>
router ospf 1<br/>
router-id 5.5.5.5<br/>
network 192.168.1.0 0.0.0.3 area 0<br/>
